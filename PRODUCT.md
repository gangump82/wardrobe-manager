# 衣橱管家 - 产品设计文档 v2

## 📱 产品概述

**产品名称：** 衣橱管家 (Wardrobe Manager)

**核心价值：** 零负担管理衣橱，AI 让穿搭变得简单

**核心痛点：**
- ❌ 传统方式：手动录入信息 → 太麻烦 → 用户放弃
- ✅ 我们方案：拍照即录入 → AI 自动识别 → 一键确认

---

## 🎯 核心功能设计

### 一、零负担录入方案

#### 1.1 拍照即录入 ⭐ 核心创新

**传统方式（❌）：**
```
拍照 → 选择类型 → 填品牌 → 填颜色 → 填尺码 → 选风格 → 标位置 → 保存
（约 2-3 分钟/件）
```

**我们的方式（✅）：**
```
拍照 → AI 自动识别 → 一键确认 → 完成
（约 10 秒/件）
```

**AI 自动识别内容：**
- ✅ 衣服类型（上衣/裤子/裙子/外套/鞋子/配饰）
- ✅ 子类型（T恤/衬衫/牛仔裤/运动裤...）
- ✅ 颜色（主色+辅色）
- ✅ 图案（纯色/条纹/格子/印花...）
- ✅ 风格标签（休闲/正式/运动/约会...）
- ✅ 适合季节
- ✅ 品牌识别（如果 Logo 清晰）

**用户只需：**
- 确认 AI 识别结果
- 可选：调整错误项

---

#### 1.2 批量拍照录入

**场景：** 刚下载 App，已有大量衣服

**流程：**
```
1. 进入批量模式
2. 连续拍照（或从相册选择）
3. AI 批量识别所有照片
4. 滑动确认/调整每件
5. 批量保存
```

**效率：** 10 件衣服约 3 分钟

---

#### 1.3 衣柜整体扫描 ⭐⭐ 创新功能

**场景：** 懒得一件件拍，直接拍整个衣柜

**流程：**
```
1. 拍摄衣柜整体照片/视频
2. AI 分析：
   - 识别每件衣服
   - 自动分类
   - 记录位置（上层/中层/下层）
3. 生成衣服清单供确认
4. 用户核对、微调
```

**技术方案：**
- 目标检测（YOLO/EfficientDet）
- 图像分割（SAM）
- 衣服分类模型
- 位置推理算法

**效果示意：**
```
原图：[衣柜照片]
     ↓ AI 分析
结果：
┌─────────────────────────┐
│ 上层：针织衫 x2, 外套 x1  │
│ 中层：T恤 x5, 衬衫 x3     │
│ 下层：牛仔裤 x4, 裙子 x2  │
└─────────────────────────┘

点击查看详情 → 照片圈出每件衣服位置
用户确认每件信息
```

### 二、位置管理优化

#### 2.1 简化位置标记

**传统方式（❌）：**
```
家 → 卧室 → 衣柜A → 第2层 → 左侧区域 → 格子3
（太精细，用户记不住）
```

**我们的方式（✅）：**

**方案一：渐进式标记**
```
首次：只标记"在哪个房间"
后续：有空再细化"在哪个柜子"
最终：AI 根据使用习惯推断
```

**方案二：粗粒度状态**
```
只需选择衣服在哪：
🏠 在衣柜（默认）
🧺 在洗衣篮
🏪 在干洗店
👔 正在穿
📦 收纳箱（换季）
```

#### 2.2 智能位置推断

**AI 自动学习：**
- 记录每次取出/放回的时间
- 分析使用习惯
- 推断衣服大概位置

**示例：**
```
用户：我的白色T恤在哪？
AI：根据记录，你上次穿是3天前，
    通常你会放回卧室衣柜上层，
    建议先去那里找找。
```

#### 2.3 找衣服神器

**功能：** 快速定位 + 提醒

```
用户：我要找那条黑色牛仔裤
App：
1. 显示最近照片
2. 显示位置提示："卧室衣柜下层"
3. 如果开启蓝牙标签 → 精确定位
4. 如果找不到 → 标记为"丢失"，
   下次录入时提醒"这不是你那条吗？"
```

### 三、激励机制设计

#### 3.1 游戏化元素

**录入激励：**
- 🏆 每录入一件 → +10 积分
- 🎯 完成衣橱清点（首次录入≥50件）→ 解锁成就
- 📸 使用衣柜扫描 → 额外奖励
- 📅 连续使用 → 签到奖励

**使用激励：**
- 👔 每次记录穿搭 → +5 积分
- 🤖 使用 AI 搭配 → +3 积分
- 📤 分享穿搭 → +10 积分

**积分兑换：**
- 解锁高级 AI 功能
- 搭配方案模板
- 主题皮肤

#### 3.2 减负设计

**原则：能不填就不填**

```
必填项：
✅ 照片（拍照自动完成）
✅ AI 识别的类型（自动）
✅ 用户确认（一键）

可选项：
⚪ 品牌（可后期补）
⚪ 价格（可后期补）
⚪ 尺码（可后期补）
⚪ 精确位置（可后期补）
```

**渐进式完善：**
- 初次录入：只填最少信息
- 穿搭记录时：提醒"要补充这件衣服的信息吗？"
- 购物记录时：自动从购物 App 导入详情

### 四、AI 功能深化

          "color": "white",

**根据用户衣橱自动建议：**

AI 分析：
- 你有 5 件白T恤，利用率很高 ✓
- 你有 2 件西装外套，从未穿过 ⚠️
- 建议搭配：西装外套 + 白T恤 + 牛仔裤

```

#### 4.3 购物建议

**智能提醒：**
```

"你已经有 5 件白T恤了，确定还要买吗？"

"你的衣橱缺少："

## 🔧 技术方案优化

### AI 识别能力

**方案一：调用云端 API**
- 百度图像识别 API
- 阿里云视觉智能 API
- 优点：准确度高、持续优化
- 缺点：需要网络、有成本

**方案二：本地模型**
- TensorFlow Lite / Core ML
- 衣服分类模型
- 优点：离线可用、无成本
- 缺点：需自行训练、占用空间

**推荐：混合方案**
```
首次识别：云端 API（高准确度）
后续使用：本地缓存 + 偶尔云端更新
离线场景：纯本地模型兜底
```

### 衣柜照片分析

**技术栈：**
- 目标检测：YOLOv8
- 图像分割：Segment Anything Model (SAM)
- 衣服分类：自定义分类模型

**流程：**
```
衣柜照片
  ↓ 目标检测
识别出 N 件衣服
  ↓ 图像分割
提取每件衣服区域
  ↓ 分类模型
判断类型/颜色/风格
  ↓ 位置推理
分析在衣柜的哪一层
  ↓ 输出
结构化的衣服列表
```
